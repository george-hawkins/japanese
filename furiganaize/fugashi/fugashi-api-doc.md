fugashi API documentation
=========================

The fugashi [README](https://github.com/polm/fugashi/blob/main/README.md) is fairly light on detail.

The following is a first pass at some documenation that was generated by Claude.

AI generated documenation
-------------------------

```
from fugashi import Tagger
tagger = Tagger()            # auto-discovers unidic, sets -r and -d for you
# tagger = Tagger('-Owakati') # extra MeCab flags go here, appended to the auto ones
```

Since you have unidic installed, Tagger() (fugashi.pyx:384-402) will:
1. Import unidic, grab DICDIR, prepend -r <…>/mecabrc -d <…> to your args.
2. Call mecab_new to build the underlying tagger.
3. Tokenize the string "日本" once to count feature fields, see 29, and set self.wrapper = UnidicFeatures29.
4. Override wrap() so every node produced from now on is a UnidicNode.

Three ways to tokenize

All three take a Python str (UTF-8 encoding is handled internally):

```
┌──────────────────────────────┬─────────────────────────────────────────────────┬─────────────────────┐
│             Call             │                     Returns                     │       Source        │
├──────────────────────────────┼─────────────────────────────────────────────────┼─────────────────────┤
│ tagger(text)                 │ list[UnidicNode] — shortcut for parseToNodeList │ fugashi.pyx:236-238 │
├──────────────────────────────┼─────────────────────────────────────────────────┼─────────────────────┤
│ tagger.parseToNodeList(text) │ list[UnidicNode], BOS/EOS stripped              │ fugashi.pyx:252-296 │
├──────────────────────────────┼─────────────────────────────────────────────────┼─────────────────────┤
│ tagger.parse(text)           │ str in MeCab's text format (depends on -O flag) │ fugashi.pyx:240-246 │
└──────────────────────────────┴─────────────────────────────────────────────────┴─────────────────────┘
```

For Python work you almost always want the first form. parse() is a thin wrapper around mecab_sparse_tostr2 and is mostly useful in -Owakati
mode when you just want space-separated surfaces as one string.

parseToNodeList silently drops the BOS/EOS sentinel nodes (fugashi.pyx:268-269), which is what you want — you get one node per real token, in
order.

The node you get back: UnidicNode

UnidicNode (fugashi.pyx:142-160) inherits everything from Node (fugashi.pyx:36-140) and adds one helper. The useful surface area:

```
for word in tagger(text):
    word.surface      # str — the token as it appeared in the input
    word.feature      # UnidicFeatures29 namedtuple — see below
    word.feature_raw  # str — raw CSV from MeCab, before splitting
    word.pos          # str — "pos1,pos2,pos3,pos4" joined (UnidicNode only)
    word.is_unk       # bool — True if MeCab couldn't find this in the dict
    word.white_space  # str — whitespace that preceded this token in the input
    word.length       # int — byte length of surface (UTF-8)
    word.rlength      # int — byte length including leading whitespace
    word.posid        # int — MeCab's internal POS id
    word.stat         # int — 0 normal, 1 unknown, 2 BOS, 3 EOS (you won't see 2/3)
```

`Node.__repr__` returns surface (fugashi.pyx:50-58), so print(word), str(word), and f"{word}" all give you the token text. That makes debugging
cheap: print(list(tagger(text))) renders as a list of surfaces.

word.feature is lazy (fugashi.pyx:72-75) — the CSV is only parsed the first time you touch it. If you only need surface, iteration stays fast.

The feature tuple: UnidicFeatures29

Defined at fugashi.pyx:31-34. It's a namedtuple, so you access fields by name or by index, and it prints nicely. The 29 fields, grouped by what
you'll actually use:

- POS hierarchy: pos1 pos2 pos3 pos4 — coarse-to-fine (名詞/動詞/…, then subtypes). Same four also available via word.pos as one string.
- Conjugation: cType (type, e.g. 五段-ラ行), cForm (form, e.g. 連用形). Blank * for non-conjugating words.
- Lemma / dictionary form: lemma is what you want if you're doing search, frequency counts, or normalization across inflections. lForm is its
reading.
- Written form: orth (as it appears), orthBase (dictionary spelling). orth ≈ surface in most cases but not always (e.g. kanji vs kana variants).
- Pronunciation: pron, pronBase — katakana, includes phonological changes. kana, kanaBase are the lemma's canonical kana.
- Word origin: goshu — 和 (native), 漢 (Sino), 外 (foreign), 混 (mixed), 固 (proper), 記号 (symbol), etc. Useful for filtering.
- Accent: aType, aConType, aModType — pitch accent, only populated for a subset of entries.
- IDs: lid, lemma_id — stable IDs into UniDic's lexicon. lemma_id groups inflected forms of the same word.

Fields that are missing in the raw CSV come through as None (fugashi.pyx:115-120), so check with if word.feature.lemma is not None.

The field name source of truth is UniDic's https://unidic.ninjal.ac.jp/faq — the top of fugashi.pyx points there.

A concrete end-to-end example

```
from fugashi import Tagger

tagger = Tagger()
text = "麩菓子は、麩を主材料とした日本の菓子。"

for word in tagger(text):
    # skip punctuation and symbols if you want
    if word.feature.pos1 == "補助記号":
        continue
    print(f"{word.surface:4}  lemma={word.feature.lemma}  pos={word.pos}  unk={word.is_unk}")
```

That pattern — iterate, filter on pos1/is_unk, collect lemma for normalized text — covers the large majority of real usage.

One subtlety worth flagging. word.surface is cached via sys.intern in a dict on the tagger (fugashi.pyx:283-285). Don't stash nodes from one
parseToNodeList call and then mutate/re-parse expecting them to stay valid for fields other than surface / white_space — the underlying
mecab_node_t* is owned by MeCab and lives only until the next parse. Extract the fields you need while iterating.
